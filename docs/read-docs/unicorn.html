I AM WHAT I CODE MAKE BLOG
유니콘 때문에 서버 입문한 썰
  2021.08.04. 수요일

  그러니까, 나는 여태까지 '서버'라는 것과 인연이 없었다. 자료구조같은 기본적인 공부와 함께 졸업에 필요한 과목들부터 순차적으로 배워나가다보니 서버에는 관심을 가질 틈도 없었다. 공부를 해야 하는 계기도, 당장의 필요도 없었던 내 관심을 서버로 끌었던 건 다른 게 아니라 분홍색 유니콘 한 마리 였다.

  작년부터 웹코딩에 관심을 가지고 방학때마다 조금씩 html과 css, javascript를 공부하고 있었다. 간단한 웹페이지를 하나 구상하고서 내 구상을 실현시킬 수 있는 코드를 구글에 찾아보는 식으로 주먹구구 공부했다. 조악하게나마 만든 웹페이지는 Github pages를 이용해서 호스팅을 하고 배포했다. 덕분에 내 Github repository는 각종 웹페이지로 채워지기 시작했다. 제대로 된 짜임새를 만들 줄 몰라 일단 내용을 떄려박기에 급급했던 첫 웹페이지에 비하면 지금 만들고 있는 이 페이지는 제법 그럴싸 해졌다고 생각한다.

  지금 만들고 있는 바로 이 웹페이지를 Github에 호스팅하고 잘 접속되는지 확인해보려는 찰나였다. 주소창에 내 웹페이지의 주소를 입력하고 Enter 키를 의기양양하게 누른 순간, 나는 처음 보는 에러와 맞닥뜨렸다. 503 Error라는 메세지와 함께 나는 사나운 유니콘과 눈이 마주쳤다.

  이미지 못지 않게 인상적이었던 에러 문구는 다음과 같다.

We're having a really bad day. The Unicorns have taken over. We're doing our best to get them under control and get GitHub back up and running.

  인터넷에 찾아보니 503 에러는 해당 페이지가 unavailable할 때 발생한다고 한다. 그럼 대체 이 에러와 멋쟁이 유니콘이 무슨 상관인 걸까. 이 기묘한 상관관계가 궁금해져 검색하다가 발견한 게 바로 깃허브 개발자의 오래된 (무려 2009년에 작성된) 블로그 글이었다. 제목은 아주 간단명료한 Unicorn!

  보아하니, 유니콘은 Ruby라는 프로그램 언어를 사용한 웹 앱을 작동시킬 수 있는 HTTP 서버의 이름인 것 같다. 해당 서버가 응답하지 않거나 오작동을 일으키면 503 에러를 뱉으면서 저 유니콘을 보여주는 것이다. 블로그 글을 쭉 읽으면서 알게된 사소한 사실인데, 502 에러가 일어났을 때에도 같은 유니콘 그림이 나타난다. 물론 문구는 503 에러와 다르다. '유니콘에게 정복당했다'는 다소 엉뚱한 503 에러 문구와 다르게 502는 친절하게 문제 상황을 설명해준다.

This page is taking too long to load. Sorry about that. Please try refreshing and contact us if the problem persists.

  블로그 글 작성자 말에 따르자면 깃허브는 30초 내로 서버의 unicorn-worker가 응답하지 않으면 그 worker를 죽이고 새 친구를 fork한다고 한다. 그 과정에서 클라이언트에게 보여지는 화면이 바로 502 에러와 유니콘인 것이다. 참으로 잔인하고 극악무도하다. 30초 내에 일을 해내지 못하면 바로 직원을 해고하는 상사라니.

  솔직하자면, 이번에 읽은 글은 이해하면서 읽었다기보다는, 아 이런 것도 있구나! 라는 깨달음을 얻으면서 읽었다고 하는 게 정확하겠다. 어떤 개념을 습득하기보다는 그 존재 자체를 알게 된 거다. 영어 단어들 자체는 어렵지 않은데 용어들이 어떤 상황과 문제와 기능을 가리키는 건지를 알 수가 없으니 이해한 내용은 거의 없다고 봐도 무방하겠다. 서버에 입문하기에 적당한 글은 아니었다. 그래서 나는 내 호기심(왜 깃허브의 503 에러 페이지에는 유니콘이 있을까?)을 해결한 데에 의의를 두며 이만 마무리하려고 한다.
